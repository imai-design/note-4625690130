1日目で「やり方」を全部見て、2日目で「自分の仕事」に落とし込む。終わったときに、現場で使える形が手元に残ります。

2日目：自分の仕事で動かす

持ってきた自分の業務データで、まる一日かけて実際の効率化を完成。最後に成果を発表します。

📺 教材について：1日目の教材は動画16本（午前のCowork編6本＋午後のCowork編2本・Code編8本）。動画はこのマニュアルの各コマに「▶ 開く」で付けてあります。視聴用アカウントは当日その場で配る24時間限定のもので、手配は本校が行います。開催が決まったら本校にご連絡ください。

進め方の基本（3つだけ覚えてください）

1

30分で1本、見て→やる

1日目は30分ごとに動画1本。見たらすぐ、自分の手で同じことを試します。

2

早い人は先に進んでOK

終わった人は次の動画へ。全員のペースを無理にそろえません。

3

最初の2本だけ全員そろえる

「Coworkの起動」と「Codeのgit導入」の2か所だけは、全員がそろうまで待ちます。

Cowork編（生活・実務） Code編（発展） ★全員そろえる 休憩
いちばん大切な運営ルール

困ったら、まず自分のClaudeに聞く

この合言葉を、初日の開会で受講者へはっきり伝えてください。質問のほとんどは、本人がClaudeに聞けば解決します。担当者は「全部教える人」ではなく「つまずきを拾う人」です。

🔑 つまずいたときの3ステップ（受講者に伝える言葉）

  1. メモに書く … 分からないことを、まず書き出す。
  2. 自分のClaudeに聞く … 「これは何？」「どうすればいい？」とそのまま聞く。英語の画面が出たら写真に撮って見せる。
  3. それでも解けなければ手を挙げる … 担当者が順番に回ります。
💡 合言葉：「誰かが詰まる所は、だいたいみんなが詰まります」。手が挙がったら、その場で全体に共有すると早く進みます。
当日までの準備

担当者が前もってやっておくこと

2日間ぶん、開催日までにこの4つを整えておきます。

1

視聴用アカウントの手配を依頼

当日配る24時間アカウントを本校に発行依頼。人数と開催日を伝えてください。

2

受講者へ事前案内を送る

下の「用意してもらうもの」を案内。とくに2日目で使う自分の仕事の実物を忘れず持参してもらいます。

3

会場とネット回線を確認

全員が同時に動画を見てダウンロードします。回線が混んでも耐えられるか確認します。

4

担当者がリハーサルする

「教える側」「受ける側」の両方で一度通します。詰まりそうな所を先に体験しておきます。

📋 受講者に用意してもらうもの：① ノートパソコン（Mac か Windows・充電器も）　② アカウント類（Claudeの有料プラン、普段使うメール・カレンダーのログイン情報）　③ 自分の仕事の実物（整理したいフォルダ、領収書、返信したいメール、マニュアルなど）。③は2日目の実践で使います。あわせてClaudeのブラウザ拡張機能を使う場面があります。
⚠ 回線は最初の山場です。動画視聴とアプリのダウンロードが重なる初日午前に最も混みます。可能なら有線、難しければ人数に余裕のある回線を。

これが2日間の核心：Code Vol.1 の中の「git導入」は、いちばん詰まりやすい所です。

このコマでやる導入の順番（VSCodeまでここで入れます）

① まず git を入れる（ターミナルで。ここが山場）
② git が入ると、アプリ版の Claude Code が使えるようになる
③ そのアプリに「Claude Code を入れて」とお願いする ← これがベース（本体）の入れ方。あとは Claude が自分でインストールを進めてくれます
④ VSCode（コードを書く・見るための画面の道具）も入れる
⑤ ここから先は Code Vol.1 の動画を見れば、ほぼそのまま進められます

受講者から聞かれそうな質問

受講者へのよくある質問（担当者向けの答え方）

受講者から聞かれたら、こう答えれば大丈夫です。多くは「自分のClaudeに聞いてみましょう」で前に進みます。

Q.パソコンが苦手でも、ついていけますか？▼
大丈夫です。動画を見て、同じことを自分の手でやるだけ。分からなくなったら、画面のClaudeにそのまま聞けます。
Q.英語の画面が出てきて分かりません。▼
画面を写真に撮って、Claudeに見せて「これは何？どうすればいい？」と聞けばOKです。日本語で教えてくれます。
Q.「許可しますか？」「警告」が出ました。押して大丈夫？▼
基本は「承認」で進めて大丈夫です。不安なときは手を挙げてください。
Q.2日目は何を持っていけばいい？▼
自分の仕事の実物（整理したいフォルダ、領収書、返信したいメール、使っているマニュアルなど）です。それを使って、本物の効率化を作ります。
Q.途中で分からなくなったら、どうすれば？▼
①メモに書く → ②自分のClaudeに聞く → ③それでも解けなければ手を挙げる。この順番でお願いします。
Q.家に帰っても使えますか？▼
使えます。当日のアカウントは24時間ぶん。続けるときは自分のアカウントに切り替えてください。
運営する側のよくある質問

担当者の疑問に答えます

Q.なぜ2日間なのですか？▼
1日目で「やり方」を全部見て、2日目でまる一日かけて「自分の仕事」に落とし込むためです。学んだだけで終わらず、現場で使える形にして持ち帰ってもらえます。
Q.2日目は何を教えればいい？▼
新しく教えることはありません。受講者が自分のデータで実践する日なので、担当者は「伴走」が中心。手が止まった人への声かけと、最後の成果発表の進行をお願いします。
Q.エンジニアでなくても運営できますか？▼
はい。教えるのは動画です。難しいのは1日目の2か所（Coworkの起動・gitの導入）だけで、その越え方もこのマニュアルに書いてあります。
Q.視聴用アカウントはどう用意しますか？▼
当日その場で配る24時間限定アカウントを使います。発行は本校が行いますので、開催日と人数を本校にお知らせください。
Q.当日トラブルが起きたら？▼
よくあるのは「アカウント作成の失敗」「契約の反映待ち」「ネットの不調」。予備のアカウントや、回線に余裕を持たせておくと安心です。困ったときは本校にご相談ください。
